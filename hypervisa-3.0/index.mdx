HyperVisa 3.0 converts entire codebases into MP4 video, achieving **22x token compression** when fed to Google Gemini's vision model. Instead of stuffing millions of text tokens into a context window, HyperVisa renders source code as bitmap frames and lets Gemini process them visually -- delivering full-repository understanding at a fraction of the token cost.

The system powers the Kijko ecosystem's code understanding layer through three interfaces: a Click CLI, a FastAPI REST API on port 8042, and an MCP server for direct LLM tool integration.

## How It Works

1. **Ingest** a codebase via `hypervisa ingest --source ./repo --name my-project`. Repomix serializes the directory into XML, HyperVisa parses and segments the output, then renders each segment as an MP4 video using the V3 renderer.

2. **Upload** each video segment to Google Gemini's file API. HyperVisa polls until the file reaches ACTIVE state, then persists the Gemini file reference in its SQLite session store alongside a CodeMap that indexes frames to source files.

3. **Query** with `hypervisa query --session my-project "How does authentication work?"`. For single-segment sessions, the query goes directly to Gemini with the video. For swarm-mode sessions, the Router selects up to 3 relevant segments, parallel agents query each one, and the Synthesizer merges their responses with conflict detection.

4. **Govern** via the Triad. Smith intercepts every tool call through the Canon rule system (reflexes block, habits warn, skills inject context). Trinity runs daily APO experiments on Smith's trace data to improve prompt quality over time.
