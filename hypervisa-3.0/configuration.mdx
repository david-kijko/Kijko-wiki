### Renderer Presets

```python title="Creating a renderer with custom config"
from hypervisa.core.hypervisa_renderer_v3 import HypervisaRendererV3, create_optimized_config

# Use the validated optimal config
config = create_optimized_config()
renderer = HypervisaRendererV3(config)

# Or use V2 presets for different use cases
from hypervisa.core.hypervisa_renderer_v2 import HypervisaRendererV2
renderer = HypervisaRendererV2.create_preset("code")    # Code-optimized
renderer = HypervisaRendererV2.create_preset("document") # Prose/docs
renderer = HypervisaRendererV2.create_preset("dense")    # Maximum density
```

## Canon Rule Configuration

Canon rules are stored as YAML files in `~/.cortex/canon/`. Each rule has the following structure:

```yaml title="~/.cortex/canon/reflexes.yaml"
- rule_id: R001            # Unique identifier
  tier: reflex             # reflex | habit | skill
  hook_target: PreToolUse  # Hook to intercept
  condition: "tool_name == 'Bash' and 'rm -rf' in input"
  action: block            # block | warn | continue | inject
  message: "R001: Destructive command blocked."
  priority: 1              # Lower number = higher priority
  enabled: true
```

### Canon Tiers

| Tier | Action | File | Purpose |
|------|--------|------|---------|
| Reflex | `BLOCK` or `WARN` | `reflexes.yaml` | Immediate prevention of dangerous operations |
| Habit | `WARN` | `habits.yaml` | Best-practice reminders logged to traces |
| Skill | `INJECT` | `skills.yaml` | Context injection at capacity thresholds |

### Default Seed Rules

<Accordions>
  <Accordion title="Reflexes (4 rules)">
    - **R001**: Block `rm -rf` commands
    - **R002**: Block `git push --force`
    - **R003**: Warn on writes to auth-related files
    - **R004**: Block `--no-verify` flag (never skip pre-commit hooks)
  </Accordion>
  <Accordion title="Habits (3 rules)">
    - **H001**: After editing, verify all callers are updated
    - **H002**: Before claiming completion, run verification commands
    - **H003**: Warn when sub-agents are spawned outside `/spec`
  </Accordion>
  <Accordion title="Skills (2 rules)">

  </Accordion>
</Accordions>

## Reward Function Weights

The 6-dimensional reward function can be tuned by modifying weights in `models/reward.py`:

| Dimension | Weight | Metric |
|-----------|--------|--------|
| `test_pass_rate` | 0.25 | Percentage of tests passing |
| `token_efficiency` | 0.20 | `1 - (tokens_used / tokens_available)` |
| `commit_fidelity` | 0.20 | `tests_added / files_modified` |
| `code_quality` | 0.15 | Linting and style compliance |
| `tool_efficiency` | 0.10 | `1 - (tool_calls / max_expected)` |
| `violation_rate` | 0.10 | `1 - (canon_violations / total_hooks)` |

## Database Configuration

HyperVisa uses SQLite with WAL mode for concurrent read access during queries.

| Setting | Value |
|---------|-------|
| Database path | `~/.hypervisa/hypervisa.db` |
| Journal mode | WAL (Write-Ahead Logging) |
| Video storage | `~/.hypervisa/videos/{session_id}/` |
| Context cache TTL | 7 days (168 hours) |
