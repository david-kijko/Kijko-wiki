The Express server at `server/routes.ts` exposes 40+ endpoints organized into 9 resource groups. Every endpoint (except `/health` and GitHub webhooks) requires authentication via Keycloak session and a specific privilege key.

## Authentication

All API requests require a valid session cookie (`kijko_docs_sid`). The `attachAuthUser` middleware runs on every request after the session is established, populating `req.authUser` from the session store.

Privilege-gated endpoints use `requirePrivilege("key")` middleware that returns `403` if the user lacks the required privilege key. Admin users (persona `admin` or client role `admin_all`) bypass privilege checks.

```typescript
// Example: only users with "pages:write" can create wiki pages
app.post("/api/wiki/pages", requirePrivilege("pages:write"), (req, res) => {
  // ...
});
```

## Health & Status

### `GET /health`

Returns server health. No authentication required.

```json
{ "ok": true, "status": { "status": "healthy", "version": "1.0.0", "uptimeSeconds": 3600 } }
```

## Wiki Pages

### `GET /api/wiki/pages`

**Privilege:** `pages:view`

List wiki pages with optional filtering and pagination.

| Parameter | Location | Type | Default | Description |
|---|---|---|---|---|
| `audience` | query | `string` | - | Filter by audience: `internal`, `external`, or `both` |
| `search` | query | `string` | - | Full-text search across title, content, and slug |
| `limit` | query | `number` | `50` | Maximum pages to return |
| `offset` | query | `number` | `0` | Pagination offset |

**Response:**
```json
{
  "pages": [
    {
      "id": 1,
      "slug": "getting-started",
      "title": "Getting Started",
      "audience": "both",
      "author": "agent",
      "wordCount": 450,
      "content": "...",
      "metadata": "{\"tags\":[\"quickstart\"]}",
      "createdAt": "2026-03-28T10:00:00.000Z",
      "updatedAt": "2026-03-28T12:00:00.000Z"
    }
  ],
  "total": 42
}
```

### `GET /api/wiki/pages/:slug`

**Privilege:** `pages:view`

Fetch a single wiki page by slug.

**Response:** `WikiPage` object or `404` if not found.

### `POST /api/wiki/pages`

**Privilege:** `pages:write`

Create a new wiki page. Request body is validated against `insertWikiPageSchema` (Drizzle-Zod).

| Field | Type | Required | Description |
|---|---|---|---|
| `slug` | `string` | Yes | Unique URL slug |
| `title` | `string` | Yes | Page title |
| `audience` | `string` | No | `internal`, `external`, or `both` (default: `internal`) |
| `author` | `string` | No | Author name (default: `agent`) |
| `wordCount` | `number` | No | Word count (default: `0`) |
| `content` | `string` | No | Markdown content (default: empty) |
| `metadata` | `string` | No | JSON metadata string (default: `{}`) |
| `createdAt` | `string` | Yes | ISO timestamp |
| `updatedAt` | `string` | Yes | ISO timestamp |

**Response:** `201` with created page, or `409` if slug already exists.

### `PUT /api/wiki/pages/:slug`

**Privilege:** `pages:write`

Update an existing page. Accepts partial `InsertWikiPage` fields.

**Response:** Updated `WikiPage` or `404`.

### `DELETE /api/wiki/pages/:slug`

**Privilege:** `pages:write`

Delete a wiki page by slug.

**Response:** `204` on success, `404` if not found.

## Repositories

### `GET /api/repos`

**Privilege:** `repos:view`

List all monitored GitHub repositories.

### `GET /api/repos/available`

**Privilege:** `repos:write`

List available GitHub repositories that can be added for monitoring. Respects the `GITHUB_ALLOWED_OWNER` policy and excludes already-monitored repos.

| Parameter | Location | Type | Description |
|---|---|---|---|
| `search` | query | `string` | Optional search filter for repo names |

### `POST /api/repos`

**Privilege:** `repos:write`

Add a GitHub repository for monitoring. Fetches the latest commit SHA and validates ownership against `GITHUB_ALLOWED_OWNER`.

| Field | Type | Required | Description |
|---|---|---|---|
| `url` | `string` (URL) | Yes | GitHub repository URL |
| `branch` | `string` | No | Branch to monitor (default: `main`) |
| `pollIntervalMinutes` | `number` | No | Polling interval in minutes (1-1440, default: 10) |

**Response:** `201` (created) or `200` (updated if already exists). Invalidates the architecture diagram cache.

### `DELETE /api/repos/:id`

**Privilege:** `repos:write`

Remove a monitored repository by ID. Invalidates the architecture diagram cache.

## Architecture

### `GET /api/architecture/diagram`

**Privilege:** `architecture:view`

Returns the current architecture diagram as a JSON graph with typed nodes (gateway, service, module, repo, infrastructure, tool, group) and labeled edges.

### `POST /api/architecture/refresh`

**Privilege:** `architecture:refresh`

Force-refresh the architecture diagram. Creates a build event recording the refresh. Returns the regenerated diagram.

## Chat

### `GET /api/chat/:sessionId`

**Privilege:** `chat:view`

Retrieve chat message history for a session.

### `POST /api/chat/:sessionId`

**Privilege:** `chat:use`

Send a chat message via REST (non-streaming). The request is validated against `chatTurnRequestSchema`.

| Field | Type | Required | Description |
|---|---|---|---|
| `content` | `string` | Yes | Message text (1-4000 chars) |
| `confirmationId` | `string` | No | ID of a pending confirmation to approve |

### `WebSocket /api/ws/chat`

**Privilege:** `chat:use`

Streaming chat interface. See [Architecture > WebSocket Chat](/docs/kijko-frontend/architecture#websocket-chat) for the full protocol specification.

## Proofshot

Proofshot provides visual verification of UI features using a headless browser.

### `GET /api/proofshot/status`

**Privilege:** `proofshot:view`

Returns current proofshot session status.

### `POST /api/proofshot/install-default`

**Privilege:** `proofshot:run`

Install default browser binaries for proofshot.

### `POST /api/proofshot/start`

**Privilege:** `proofshot:run`

Start a proofshot verification session.

| Field | Type | Required | Description |
|---|---|---|---|
| `description` | `string` | No | Session description (max 200 chars) |
| `url` | `string` | No | Target URL to verify |
| `run` | `string` | No | Test script content (max 2000 chars) |
| `port` | `number` | No | Port to target (1-65535) |
| `headed` | `boolean` | No | Run browser in headed mode (default: false) |

### `POST /api/proofshot/stop`

**Privilege:** `proofshot:run`

Stop the running proofshot session.

### `GET /proofshot/artifacts/*`

**Privilege:** `proofshot:view`

Serve proofshot artifact files (screenshots, videos, reports).

### `GET /api/swarm/health`

## Admin

### `GET /api/admin/users`

**Privilege:** `admin:users`

List all managed Keycloak users with their persona profiles and privileges.

### `POST /api/admin/users`

**Privilege:** `admin:users`

Create a managed user in Keycloak. Assigns persona profile and privilege keys.

### `PUT /api/admin/users/:id`

**Privilege:** `admin:users`

Update a managed user. Cannot update yourself. Cannot demote/disable the last admin.

### `DELETE /api/admin/users/:id`

**Privilege:** `admin:users`

Delete a managed user. Cannot delete yourself. Cannot delete the last admin.

### `POST /api/admin/users/:id/reset-password`

**Privilege:** `admin:users`

Reset a user's password. Minimum 8 characters. Cannot reset your own password through this endpoint.

## GitHub Webhooks

### `POST /api/webhook/github` (and `/webhook/github`)

No authentication required (uses webhook signature verification).

Receives GitHub push webhooks. Verifies the `x-hub-signature-256` header against `GITHUB_WEBHOOK_SECRET`, then creates a build event.

<Callout type="info">
Both `/api/webhook/github` and `/webhook/github` paths are registered for backwards compatibility with existing webhook configurations.
</Callout>

## Auth Routes

### `GET /api/auth/login`

Initiates the Keycloak OIDC login flow. Generates PKCE challenge, stores state in session, and redirects to Keycloak's authorization endpoint.

### `GET /api/auth/callback`

Handles the OIDC callback. Exchanges the authorization code for tokens, fetches user info, derives privileges, and stores the session.

### `GET /api/auth/me`

Returns the current authenticated user's profile, privileges, and navigation catalog. Returns `401` if not authenticated.

### `POST /api/auth/logout`

Destroys the session and optionally redirects to Keycloak's end-session endpoint.

### `POST /api/auth/refresh`

Force-refresh the access token using the stored refresh token. Returns the updated user profile.
