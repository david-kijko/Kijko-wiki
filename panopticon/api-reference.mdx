Panopticon exposes four API surfaces: a REST API for dashboard integration, an MCP server for AI agent tool access, a CLI for operators, and a Nexus service for cross-namespace Temporal workflows. This reference documents every endpoint, tool, command, and operation with parameters, examples, and error handling.

## REST API

Flask API on port 8043 (configurable via `PANOPTICON_API_PORT`). CORS is enabled for all origins on `/api/*` paths.

### GET /api/panopticon/health

System health check including Temporal connectivity, MCP server count, tool count, and catalog status.

**Response:**

```json
{
  "status": "ok",
  "service": "panopticon",
  "version": "0.1.0",
  "temporal": {
    "address": "localhost:7233",
    "connected": true
  },
  "servers_count": 3,
  "tools_count": 47,
  "catalog_built": true,
  "catalog_updated": "2026-03-29T10:15:00"
}
```

### GET /api/panopticon/servers

List all registered MCP servers with their runtime status.

**Response:**

```json
[
  {
    "name": "firecrawl",
    "command": "npx",
    "args": ["@anthropic/firecrawl-mcp"],
    "enabled": true,
    "running": true,
    "status": "connected"
  },
  {
    "name": "exa",
    "command": "npx",
    "args": ["@anthropic/exa-mcp"],
    "enabled": true,
    "running": false,
    "status": "standby"
  }
]
```

Status values: `connected` (process running), `standby` (enabled but not running -- stdio servers start on-demand), `disabled` (explicitly disabled in config).

## MCP Server Tools

The MCP server (`panopticon serve`) exposes 7 tools via stdio JSON-RPC. These are the tools that AI agents call through the MCP protocol.

### health_check

Check Panopticon status: Temporal connection, registered MCP servers, catalog freshness, and API key pool summary.

**Parameters:** None

**Example Response:**

```json
{
  "temporal": {"address": "localhost:7233", "connected": true},
  "servers": {
    "firecrawl": {"enabled": true, "command": "npx"}
  },
  "catalog": {"built": true, "schema_count": 47, "updated_at": "2026-03-29T10:15:00"},
  "total_tools": 47,
  "key_pool": {"initialized": true, "pool_size": 3, "active_keys": 3}
}
```

### browse_tools

Search available tools using natural language. Uses HyperVisa video catalog + Gemini visual search. Returns top-3 matches with confidence scores. Output: ~500 tokens.

<TypeTable
  type={{
    query: { type: 'string', description: 'Natural language description of what you want to do', required: true },
  }}
/>

**Example:**

```json
// Input
{"query": "scrape a webpage"}

// Output
[
  {"server": "firecrawl", "tool": "scrape", "description": "Scrape a URL", "confidence": 0.95, "reason": "Direct match for web scraping"},
  {"server": "exa", "tool": "get_contents", "description": "Get URL contents", "confidence": 0.72}
]
```

### get_tool_details

Get the full JSON schema for a specific tool. Use after `browse_tools` to see parameters. Output: ~2,000 tokens.

<TypeTable
  type={{
    tool_name: { type: 'string', description: 'Tool name (e.g., "scrape") or server.tool (e.g., "firecrawl.scrape")', required: true },
  }}
/>

### execute_tool

Execute a tool call directly (raw mode). Routes through Temporal for durable execution.

<TypeTable
  type={{
    server: { type: 'string', description: 'MCP server name', required: true },
    tool: { type: 'string', description: 'Tool name on the server', required: true },
    params: { type: 'object', description: 'Tool parameters' },
  }}
/>

### smart_execute

Natural language to tool to result. Gemini finds the right tool, fills parameters, executes via Temporal, returns result.

<TypeTable
  type={{
    intent: { type: 'string', description: 'What you want to accomplish (e.g., "fetch the content of example.com")', required: true },
  }}
/>

### execute_parallel

Execute multiple tool calls with a dependency graph. Independent calls run in parallel. Supports compensating rollback on failure.

<TypeTable
  type={{
    steps: { type: 'array', description: 'Array of step objects with server, tool, params, depends_on, and compensate fields', required: true },
  }}
/>

**Step Schema:**

```json
{
  "server": "github",
  "tool": "create_branch",
  "params": {"repo": "my-repo", "branch": "feature-x"},
  "depends_on": [],
  "compensate": {
    "tool": "delete_branch",
    "params": {"branch": "feature-x"}
  }
}
```

### key_pool_health

Detailed Gemini API key pool status. Shows per-key RPM usage, cooldown state, failure counts, and key type (regular vs Vertex).

**Parameters:** None

**Example Response:**

```json
{
  "pool_size": 3,
  "active_keys": 3,
  "cached_clients": 2,
  "keys": [
    {"key_suffix": "...Xy4z", "key_type": "regular", "status": "available (3/10 rpm)", "total_requests": 45},
    {"key_suffix": "...Ab2c", "key_type": "vertex", "status": "cooling (23s)", "total_requests": 12}
  ]
}
```

## CLI Commands

The CLI (`panopticon`) is built with Click 8.0+.

### panopticon discover

Build or refresh the HyperVisa video catalog from MCP servers.

```bash
panopticon discover --rebuild
```

| Flag | Description |
|------|-------------|
| `--rebuild` | Force rebuild catalog from all servers (ignore cache) |

### panopticon browse

Search available tools by natural language query.

```bash
panopticon browse "web scraping"
```

### panopticon details

Get full JSON schema for a tool.

```bash
panopticon details firecrawl.scrape
```

### panopticon exec

Execute a tool call (raw mode). Use `server.tool` notation.

```bash
panopticon exec firecrawl.scrape '{"url":"https://example.com"}'
```

### panopticon ask

Smart execution: natural language to tool to result.

```bash
panopticon ask "fetch the content of example.com"
```

### panopticon health

Check system health: Temporal, MCP servers, catalog.

```bash
panopticon health
```

### panopticon worker

Start the Temporal worker process. Must be running for any workflow execution.

```bash
panopticon worker
```

### panopticon serve

Start the Panopticon MCP server (stdio transport).

```bash
panopticon serve
```

### panopticon api

Start the REST API server for dashboard integration.

```bash
panopticon api --port 8043
```

### panopticon register

Register a new MCP server.

```bash
panopticon register firecrawl npx -a @anthropic/firecrawl-mcp -e FIRECRAWL_API_KEY=fc-xxx
```

| Flag | Description |
|------|-------------|
| `-a`, `--args` | Server arguments (repeatable) |
| `-e`, `--env` | Environment variables as KEY=VALUE (repeatable) |

## Nexus Operations

The Nexus service (`panopticon/temporal/nexus_service.py`) exposes 4 operations for cross-namespace invocation via Temporal Nexus.

### execute_tool

**Input:** `ExecuteToolInput(server_name, tool_name, params)`
**Output:** `ExecuteToolOutput(tool, result, error, duration_ms, ok, retried)`
**Backing Workflow:** `ToolExecutionWorkflow`

### browse_tools

**Input:** `BrowseToolsInput(query, max_results=3)`
**Output:** `BrowseToolsOutput(tools, query, ok, error)`
**Backing Workflow:** `BrowseToolsWorkflow`

### refresh_catalog

**Input:** `RefreshCatalogInput(force=False)`
**Output:** `RefreshCatalogOutput(video_path, file_name, file_uri, schema_count, ok, error)`
**Backing Workflow:** `CatalogRefreshFromRegistryWorkflow`

### health_check

**Input:** `HealthCheckInput()`
**Output:** `HealthCheckOutput(connected, servers_running, servers_total, tools_count, agents, temporal_connected)`
**Backing Workflow:** `HealthCheckWorkflow`

## Error Handling

All MCP tool calls return `CallToolResult` with `isError` set to `true` on failure. The error message is in the `TextContent` payload.

All REST endpoints return JSON. Errors follow this pattern:

```json
{
  "error": "error description"
}
```

Common error patterns:

| Error | Cause | Resolution |
|-------|-------|-----------|
| `"No catalog built. Run: panopticon discover --rebuild"` | Video catalog not yet created | Run `panopticon discover --rebuild` |
| `"Tool not found: X"` | Tool name does not exist in any server | Check spelling, run `panopticon browse` to find correct name |
| `"All keys rate-limited"` | Gemini key pool exhausted | Add more keys or wait for cooldown |
| `"Unknown MCP server: X"` | Server not registered | Register with `panopticon register` |
| `"Server X is disabled"` | Server explicitly disabled in config | Set `enabled: true` in `servers.yaml` |
