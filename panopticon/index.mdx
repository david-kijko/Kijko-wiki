Panopticon is the central nervous system for an AI agent fleet. It provides token-efficient tool discovery through HyperVisa video encoding, crash-safe tool execution via Temporal workflows, and collective learning across agent sessions through a hive mind memory system. Where traditional MCP setups dump all tool schemas into context (~50,000+ tokens), Panopticon achieves a ~98% context token reduction by rendering schemas as 1fps MP4 video and using Gemini visual search to return only the top-3 matches (~500 tokens).

The system sits between AI agents (Claude Code, Oracle, Smith, Trinity) and downstream MCP servers (Firecrawl, Exa, Graphiti, NotebookLM, and others). Every tool call flows through Temporal for durable execution -- if a worker crashes mid-call, Temporal replays from the last checkpoint and the call completes without data loss.

## How It Works

Panopticon operates on a three-tier progressive disclosure model that minimizes context token usage while maximizing tool discovery accuracy:

1. **Browse** -- An agent sends a natural language query (e.g., "scrape a webpage"). Gemini watches the HyperVisa video catalog and returns the top-3 tool matches with confidence scores. Cost: ~500 tokens.
2. **Detail** -- The agent requests the full JSON schema for the selected tool. This provides complete parameter definitions without loading schemas for tools that were not needed. Cost: ~2,000 tokens.
3. **Execute** -- The tool call routes through a Temporal `ToolExecutionWorkflow`. If the downstream MCP server crashes, Temporal restarts it and retries the call from the checkpoint. The agent gets a result or a definitive error.

Every execution generates a trace that feeds into the reinforcement learning pipeline (Agent Lightning). Over time, Panopticon learns which tools work best for which query types and promotes successful patterns to the shared hive mind.
