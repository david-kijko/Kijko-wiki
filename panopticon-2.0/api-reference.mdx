All types are defined in `apps/agent/src/lib/pipeline/types.ts` and exported from `apps/agent/src/lib/pipeline/index.ts`. The Mastra tool schemas are in `apps/agent/src/mastra/tools/pipeline-tools.ts`.

## Core Interfaces

### PipelineAgent

The interface that every agent implements:

```typescript
interface PipelineAgent {
  name: string;
  execute(context: AgentContext): Promise<AgentResult>;
}
```

### AgentContext

The context object passed to every agent. It carries MCP data, previous phase results, and pipeline metadata:

```typescript
interface AgentContext {
  /** Unique run identifier. */
  pipelineId: string;
  /** Repository slug being documented. */
  repoSlug: string;
  /** Local filesystem path to the repo checkout. */
  repoPath: string;
  /** Existing wiki pages (slug list) for gap analysis. */
  existingPages: string[];
  /** CodeGraph symbol data, if available. */
  codeGraphData?: CodeGraphData;
  /** CGC analysis data, if available. */
  cgcData?: CgcData;
  /** NotebookLM research context, if available. */
  notebookContext?: NotebookData;
  /** Forensic-ingest artifacts (Phase 0), if available. */
  forensicIngest?: ForensicIngestArtifacts;
  /** Current pipeline state (for reading previous phase results). */
  pipelineState: PipelineState;
  /** MCP tool availability flags. */
  toolAvailability: ToolAvailability;
}
```

### AgentResult

The return value from every agent's `execute()` method:

```typescript
interface AgentResult {
  /** Whether the agent completed successfully. */
  success: boolean;
  /** Structured output data (varies per agent). */
  artifacts: Record<string, unknown>;
  /** Human-readable error messages. */
  errors: string[];
  /** Execution metrics. */
  metrics: AgentMetrics;
  /** Optional context additions for downstream phases. */
  nextContext?: Partial<AgentContext>;
}

interface AgentMetrics {
  durationMs: number;
  llmCalls: number;
  toolCalls: number;
}
```

### PipelineConfig

Configuration for a pipeline run:

```typescript
interface PipelineConfig {
  repoSlug: string;
  repoPath: string;
  triggerType: TriggerType;
  /** Phases to skip (merged with trigger-type defaults). */
  skipPhases?: PhaseName[];
  /** When true, log actions but don't write files. */
  dryRun?: boolean;
  /** Specific page slugs to scope work to. */
  targetSlugs?: string[];
}

type TriggerType =
  | "full"
  | "incremental"
  | "single_page"
  | "preview"
  | "drift_check"
  | "manual";
```

## State Types

### PipelineState

Top-level state for a pipeline run:

```typescript
interface PipelineState {
  id: string;
  repoSlug: string;
  currentPhase: PhaseName | null;
  phases: PhaseState[];
  startedAt: string;
  completedAt?: string;
  status: "queued" | "running" | "completed" | "failed" | "paused";
}
```

### PhaseState

Per-phase state within a pipeline run:

```typescript
type PhaseStatus = "pending" | "running" | "complete" | "failed" | "skipped";

interface PhaseState {
  name: PhaseName;
  status: PhaseStatus;
  agentResult?: AgentResult;
  error?: string;
  startedAt?: string;
  completedAt?: string;
}
```

### Phase Constants

```typescript
const PHASE_NAMES = [
  "audit", "ia", "writer", "verifier", "qa", "monitor", "publish",
] as const;

type PhaseName = (typeof PHASE_NAMES)[number];

const PHASE_ORDER: Record<PhaseName, number> = {
  audit: 0, ia: 1, writer: 2, verifier: 3, qa: 4, monitor: 5, publish: 6,
};
```

## MCP Data Shapes

These types define the data returned by MCP client pre-fetches and stored in `AgentContext`:

### CodeGraphData

```typescript
interface CodeGraphSymbol {
  name: string;
  kind: "function" | "class" | "interface" | "type" | "variable" | "method";
  filePath: string;
  visibility: "exported" | "internal";
  signature?: string;
}

interface CodeGraphData {
  symbols: CodeGraphSymbol[];
  callGraph: Array<{ caller: string; callee: string }>;
}
```

### CgcData

```typescript
interface CgcData {
  deadCode: Array<{ symbol: string; file: string }>;
  complexity: Array<{ file: string; score: number }>;
  dependencies: Array<{ from: string; to: string }>;
}
```

### NotebookData

```typescript
interface NotebookData {
  summaries: Array<{ topic: string; content: string }>;
  sourceIds: string[];
}
```

### ForensicIngestArtifacts

```typescript
interface ForensicSourceFile {
  /** Cluster/subsystem name (e.g. "apps-agent-src-lib-1"). */
  name: string;
  /** Absolute path to the generated markdown file. */
  path: string;
  /** Markdown content of the source. */
  content: string;
}

interface ForensicIngestArtifacts {
  /** Per-subsystem source markdown documents from segmentation. */
  sources: ForensicSourceFile[];
  /** Full dependency graph JSON (clusters, edges, PageRank, cycles). */
  dependencyGraph: Record<string, unknown>;
  /** Code Map Index markdown (if generated). */
  codeMapIndex?: string;
}
```

### ToolAvailability

```typescript
interface ToolAvailability {
  codegraph: boolean;
  cgc: boolean;
  notebooklm: boolean;
  repomix: boolean;
  /** True if forensic-ingest Python scripts are available. */
  forensicIngest: boolean;
}
```

## PipelineStateStore Interface

```typescript
interface PipelineStateStore {
  save(state: PipelineState): void;
  load(id: string): PipelineState | null;
  list(): PipelineState[];
}
```

The default implementation is `InMemoryStateStore`. See [State Persistence](/docs/panopticon-2.0/state-persistence) for details on implementing custom backends.

## PipelineOrchestrator API

```typescript
class PipelineOrchestrator {
  constructor(deps: OrchestratorDeps);

  /** Execute a full pipeline run. */
  execute(config: PipelineConfig): Promise<PipelineState>;

  /** Resume a failed pipeline from the last successful phase. */
  resume(pipelineId: string, config: PipelineConfig): Promise<PipelineState>;

  /** Get the current state of a pipeline. */
  getStatus(pipelineId: string): PipelineState | null;

  /** List all pipeline runs. */
  listRuns(): PipelineState[];
}

interface OrchestratorDeps {
  mastraAgent?: { generate: (prompt: string) => Promise<{ text: string }> };
  listExistingPages: () => string[];
  stateStore?: PipelineStateStore;
}
```

## Mastra Tool Schemas

The pipeline exposes three Mastra tools at `apps/agent/src/mastra/tools/pipeline-tools.ts`:

### classify-page-type

Classifies a wiki page into one of 10 page types based on slug and optional content analysis.

**Input**:
```typescript
{
  slug: string;       // Required -- page slug to classify
  content?: string;   // Optional -- page content for accuracy
}
```

**Output**:
```typescript
{
  pageType: PageType;             // e.g. "concept_explainer"
  confidence: number;             // 0.0 to 1.0
  suggestedSections: string[];    // e.g. ["Overview", "Why It Matters", ...]
}
```

### validate-mdx

Validates MDX content for structural correctness (frontmatter, heading hierarchy, JSX balance).

**Input**:
```typescript
{
  mdx: string;  // The MDX content to validate
}
```

**Output**:
```typescript
{
  valid: boolean;
  errors: Array<{
    type: string;      // "frontmatter" | "heading" | "jsx"
    message: string;
    line?: number;
  }>;
}
```

### run-pipeline

Triggers a pipeline run and returns summary results.

**Input**:
```typescript
{
  repoSlug: string;                   // Required
  triggerType: TriggerType;           // Default: "manual"
  dryRun?: boolean;                   // Default: false
}
```

**Output**:
```typescript
{
  pipelineId: string;
  status: string;
  phasesCompleted: number;
  totalPhases: number;
  errors: string[];
}
```
