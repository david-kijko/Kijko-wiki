# Kijko Wiki

Version-controlled wiki pages for all Kijko repositories. Autonomous documentation powered by WikiAgent.

## Structure

```
Kijko-wiki/
├── panopticon/          # Central monitoring & observability platform
├── panopticon-2.0/      # Durable agent orchestration layer
├── hypervisa-3.0/       # Video-mediated context engine
├── kijko-frontend/      # Next.js docs reader (Fumadocs + CopilotKit)
├── baton-exchange/      # Context relay protocol for agent handoffs
├── _shared/             # Cross-repo index, getting-started, style system
└── wiki-research/       # SuperMemory reference analysis, templates, gap reports
```

## How It Works

1. **Source repos** push to `main` -> GitHub Actions triggers wiki rewrite
2. **WikiAgent pipeline** (Audit -> IA -> Writing -> Verification -> QA -> Monitor) generates/updates MDX pages
3. **Kijko-wiki repo** receives the updated pages via commit
4. **docs.kijko.nl** wiki tab renders the latest content via Fumadocs

## Quality Target

Wiki pages target the depth and structure of [SuperMemory docs](https://supermemory.ai/docs/intro):
- 9+ page type templates (Landing, Concept, Quickstart, Integration, API Reference, Cookbook, etc.)
- Full architecture deep-dives with Mermaid diagrams
- API reference with interactive examples
- Configuration guides with parameter tables
- Deployment and operations guides

## Content Format

All wiki pages are authored in MDX with Fumadocs frontmatter. Legacy `.md` versions are preserved for reference.

## Contributing

Wiki content is primarily generated by the WikiAgent autonomous pipeline. Manual edits are welcome — the agent will preserve human modifications during rewrites.
